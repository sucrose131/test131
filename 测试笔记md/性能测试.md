# 一、什么是性能测试

 

性能测试是通过自动化的测试工具模拟多种正常、峰值以及异常负载条件来对系统的各项性能指标进行测试

我们可以认为性能测试是：通过在测试环境下对系统或构件的性能进行探测，用以验证在生产环境下系统性能是否达到预估的性能需求，发现系统可能存在的性能瓶颈，进而改善优化并系统的性能，提高系统的可扩展性、稳定性。

 

负载测试（Load Testing）：目标是测试在不同负载情况下的系统性能情况。常从比较小的负载开始，逐渐增加并发数、数据量， 观察并记录不同负载情况下系统的各项指标，直至系统无法承受而崩溃或服务不可用。

压力测试（Stress Testing）：在超强的、近乎会导致系统崩溃的负荷(大数据量、大量并发用户等)下，观察系统的cpu利用率，内存使用率，磁盘I/O吞吐率，网络吞吐量等的变化。

稳定性测试 ：持续处理压力请求的能力，7*24小时

 

# 二、系统用户数、在线用户数、并发用户数

 

### 1、系统用户数

系统用户数是某个系统拥有的实际用户，是数据库存在的数据数量；

比如：QQ 注册了 300万人 那么QQ的系统用户数就是300万



### 2、在线用户数

在线用户数，是同时登录系统在线的用户数量，但是在线用户数不对服务器造成压力，因为没有对某项功能点进行并发访问；

 

### 3、并发用户数

并发用户数就是对某一操作或者功能，进行同时访问的用户数量。 是真正对服务器产生压力的。

 

# 三、性能测试指标

 

重要指标

软件：TPS，响应时间，响应错误率，

服务器：CPU, 内存，网络(带宽)

 

### 1.TPS（Transaction per second）

吞吐率是指单位时间内系统能处理的请求数量，体现系统处理请求的能力，这是目前最常用的性能测试指标

​	每秒钟系统能够处理的交易或者事务的数量.它是衡量系统处理能力的重要指标.

 

### 2.平均事务响应时间（Average Transation Response Time ）

很好：<2s

较好：2-5s

一般：5-8s

不能接受：>8s

 

### 3.CPU利用率（Processor/%Processor Time ）

好：<75%

坏：>85%

瓶颈：持续95%

### 4.剩余的可用内存（Memory / Avaiable Mbytes ）

好：剩余50%以上

坏：剩余20%

瓶颈：剩余10% 

我们更加关心的是服务器并发处理能力的上限，也就是单位时间内服务器能够处理的最大请求数，即最大吞吐率。

所以我们普遍使用“压力测试”的方法，通过模拟足够多数目的并发用户，分别持续发送一定的HTTP请求，并统计测试持续的总时间，计算出基于这种“压力”下的吞吐率，即为一个平均计算值

### **1、利用已有.jtl文件生成报告**

如果已经有经过测试生成的.jtl文件，可以利用该文件直接生成HTML可视化测试报告。

进入jmeter的bin目录下，输入如下命令：

jmeter -g kouyi.jtl -o /path

 -g：后跟kouyi.jtl文件所在的路径

 -o：后跟生成的HTML文件存放的路径

例./jmeter -g D:\jmeter\jmetewenjian\rou.jtl -o D:\jmeter\xingnengwendang

./  ：powershell里需要绝对路径

**PS：**如果是在Windows环境命令行运行，必须指定生成的HTML文件存放文件夹，否则会报错；如果是linux环境，如指定路径下不存在该文件夹，会生成对应的文件夹存放报告文件！



### 2、**无.jtl文件生成测试报告**

jmeter -n -t **test.jmx**(脚本的绝对路径) -l **result.jtl**(自定义的名称) -e -o **\tmp\result_report**(测试报告的绝对路径)

例：jmeter -n -t C:\Users\ZC\Desktop\test.jmx -l C:\Users\ZC\Desktop\report.jtl -e -o C:\Users\ZC\Desktop\result

 参数说明:

)n：非GUI模式执行JMeter；
)t： 脚本文件(.jmx文件)的路径；
)l： 指定生成测试结果的保存文件(.jtl格式)，此文件必须不存在；
)e：测试结束后，生成测试报告；
)o：用于存放测试报告的路径；



例：./jmeter -n -t F:\apache-jmeter-5.6.3\jmetewenjian\xingnengceshi.jmx -l F:\apache-jmeter-5.6.3\jmetewenjian\report.jtl -e -o F:\apache-jmeter-5.6.3\jmetewenjian\test
