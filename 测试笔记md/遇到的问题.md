问题1.

元素隐藏后，设置等待会导致定位不到元素。

在 XPath 或 CSS 选择器中，使用相对路径而不是绝对路径。相对路径可以避免因元素位置变化而导致的定位失败。

元素定位不到时，可通过其他元素推测出这个元素的位置

对于动态生成的 `id` 或 `class`，可以使用部分匹配的方法来定位元素。例如，使用 `contains` 函数。

```
element = driver.find_element_by_xpath("//div[contains(@id, 'dynamic_part')]")
```

有时候一个class可能包含多个元素，可以采用XPATH加class加文本的方式定位比如

其中是否加contains表示是否为模糊匹配比如下方是精准匹配class和模糊匹配text

```
Media_list = (By.XPATH, "//div[@class='label']//span[contains(text(), '频道')]")#新闻媒体列表
```

在 HTML 中，元素的 `class` 属性值是由多个类名组成的字符串，类名之间用空格分隔。例如：

```
<div class="el-select-dropdown__item is-hovering">采集</div>
```

这里的 `class` 属性值是 `"el-select-dropdown__item is-hovering"`，它包含两个独立的类名：`el-select-dropdown__item` 和 `is-hovering`。



而 XPath 的 `contains` 函数会查找整个属性值中是否包含指定的字符串。如果你使用 `contains(@class, 'el-select-dropdown__item is-hovering')`，XPath 会严格匹配包含 `"el-select-dropdown__item is-hovering"` 这个**连续字符串**的元素。这意味着：

- 如果类名顺序改变（例如 `class="is-hovering el-select-dropdown__item"`），将无法匹配。
- 如果元素只有其中一个类名（例如 `class="el-select-dropdown__item"`），也无法匹配。

### 正确的定位方法

更可靠的做法是分别检查每个类名是否存在：

```
News_source_collect= (By.XPATH,"//div[contains(@class, 'el-select-dropdown__item') and contains(@class, 'is-hovering')]")
```

```
def get_all_keywords_on_the_current_page(self,locator):
    """获取当前页所有关键词"""
    by, value = locator
    elements = self.find_elements(by, value)
    titles = [el.text.strip() for el in elements if el.text.strip()]
    return titles
```
