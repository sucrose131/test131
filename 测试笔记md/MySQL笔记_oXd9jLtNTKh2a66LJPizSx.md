# MySQL笔记

# 基础知识

数据库：管理数据的仓库

关系型数据库：采用了关系模型来组织数据的数据库。关系模型就是二维表格模型。适用于不经常变动格式的数据

非关系型数据库：又称为NoSQL（not only SQL)。最普遍的定义是“非关联型”的，强调key-value的方式存储数据。常见的有MongoDB、Redis。适用于经常变动格式的数据

MySQL优点

*   成本低：开放源代码，社区版免费，企业版收费

*   性能高：执行很快

*   简单：很容易安装和使用

*   可以安装在不同的操作系统，并且提供多种编程语言的接口

常见缩写

| DB   | database ，数据库，有组织（能够增删该查）的存储数据               |
| ---- | -------------------------------------------- |
| DBMS | database management system，数据库管理系统，即管理数据库的软件 |
| DBA  | Database Administrator，数据库管理员                |
| SQL  | Structured Query Language， 结构化查询语言           |
| DQL  | 数据查询语言（数据的查询）                                |
| DML  | 数据操作语言（数据的增、删、改）                             |
| DDL  | 数据定义语言（库和表的管理）                               |

常见单词

varchar：Variable Character ，可变字符串（会随字符长度自动分配空间）

record：原义记录。在表中代表一行的数据

feild：原义领域。在表中代表一列的数据，也叫字段

enum：enumerate，枚举

timestamp：stamp邮票，标记。timestamp表示时间戳。

时间戳是使用数字签名技术产生的数据，签名的对象包括了原始文件信息、签名参数、签名时间等信息。时间戳系统用来产生和管理时间戳，对签名对象进行数字签名产生时间戳，以证明原始文件在签名时间之前已经存在。

collate：核对，校对

decimal：十进制的，小数的

ambiguous：有歧义的，模棱两可的

慢查询：查询时响应时间慢

5.  注释

    单行注释：#注释

    &#x20;            \-- 注释（注意注释前面有空格）

    多行注释：/\* 注释 \*/

## 加号 +

只有一个运算符的作用

情况

*   两个操作数都为数值型，则做加法运算

*   有一方或两方都为字符型，则试图将字符型转换成数值型

    *   如果转换成功，则继续做加法运算。如 '2'+3 的结果为5

    *   如果转换失败，则将字符型数值转换成0 。如 'a'+3 的结果为3

*   只要有一方为null ，则结果肯定为null

# 增加

## 增加数据库&#x20;

常用

```sql
create database 数据库名 character set utf8;
```

一般格式

```sql
CREATE DATABASE [IF NOT EXISTS] <数据库名> [CHARACTER SET <字符集名>] [COLLATE <校对规则名>];
```

*   \[ ]中的内容是可选的

*   <数据库名>：创建数据库的名称。命名规则：**不能以数字开头，不区分大小写**。

*   IF NOT EXISTS：MySQL 不允许在同一系统下创建两个相同名称的数据库，如果创建同名数据库会报错。加上此语句后，只有该数据库不存在时才会创建；如果已存在，直接忽略，不会报错

*   CHARACTER SET：指定数据库的字符集。指定字符集的目的是为了避免在数据库中存储的数据出现乱码的情况。如果在创建数据库时不指定字符集，那么就使用系统的默认字符集。

*   COLLATE：指定字符集的默认校对规则。

### 校对规则

是在字符集内用于字符比较和排序的一套规则，一般以\_ci结尾的忽略大小写，\_cs结尾的大小写敏感

增加字段就是在修改表

## 增加数据

方法1：不写字段名，按顺序写出所有值，默认值写成default，想空着的写成null（字段允许为空才行），不能什么都不写。支持一次增加多条数据

```sql
INSERT INTO 表名 VALUES(值a1,···,值an),(值b1,···,值bn),···;
```

方法2：表名后写上字段名，给指定字段赋值，其余字段为默认值，可以不按顺序，但后面的值要和前面的字段一一对应

```sql
insert into 表名(字段1,···,字段n) value(值1,···,值n);
```

方法3：用set给指定字段赋值，其余字段为默认值，可以不按顺序。只能一次增加一条数据

```sql
insert into 表名 set 字段1=字段值1,···,字段n=字段值n;
```

# 删除

## 删除数据库

常用

```sql
drop database 数据库名;
```

一般格式

```sql
DROP DATABASE [ IF EXISTS ] <数据库名>;
```

*   IF EXISTS：如果要删除的数据库不存在，会报错，加入此语句后，当数据库不存在时则忽略，不会报错

*   删除数据库时会删除其中的所有表格和表中数据，需要获得数据库 DROP 权限

*   MySQL 安装后，系统会自动创建名为 information\_schema 和 mysql 的两个系统数据库，系统数据库存放一些和数据库相关的信息，如果删除了这两个数据库，MySQL 将不能正常工作

## 删除表

```sql
DROP TABLE [IF EXISTS] 表名1 [ ,表名2, 表名3 ...];
```

数据表之间经常存在外键关联的情况，这时如果直接删除父表，会破坏数据表的完整性，也会删除失败。 &#x20;

删除父表有以下两种方法： &#x20;

*   先删除与它关联的子表，再删除父表；但是这样会同时删除两个表中的数据

*   将关联表的外键约束取消，再删除父表；适用于需要保留子表的数据，只删除父表的情况

## 删除字段

删除字段就是在修改表

## 删除数据

```sql
DELETE FROM 表名 [WHERE 子句] [ORDER BY 子句] [LIMIT 子句];
```

清空表中所有数据，不能加筛选条件

```sql
TRUNCATE [TABLE] 表名;

```

区别

*   用delete删除全部数据后，再增加数据，自增长字段（常见的有id）的值从断点处开始，而truncate清空数据后，自增长字段的值从1开始

*   delete删除可以回滚，truncate删除不能回滚

# 字段查询

## 排序查询

order by

语法：select 字段1 from 表名 \[where 筛选条件] order by 字段2 \[asc或desc],字段3 \[asc或desc]

asc为ascend（上升的缩写），表示升序排列，默认升序排列

desc为descend（下降） 的缩写，表示降序排列

*   order by 可以支持单个字段，多个字段，表达式，函数，别名

*   order by一般放除limit语句外的最后面

多个排序条件用逗号分隔，表示先按前面的字段排序，当前面的字段相同时，再按后面的字段排序

升序排序规则（降序相反）

1.  null 在最前

2.  空字符在有字符的前面

3.  负数在正数前面（因为越小的越前）

order by 整数n：按查询到的第n个字段排序，从1开始，超出索引会报错

### 聚合函数

又称统计函数、分组函数、组函数，输入多个参数，返回一个。主要做统计用

| ------- | -------------- |

统计总行数 select count(\*) from 表名;

## 分组查询

使用聚合函数后才需要分组查询，没有用聚合函数使用分组查询是没有意义的

语法 select 聚合函数,列（要求出现在group by后面） from 表名 \[where 筛选条件] group by 分组列表 order by 子句

分组查询后再筛选 group by 后用 having

with rollup返回聚合函数对所有数据的值在group by语句后面，having前面，并且不受having条件的影响

使用with rollup需注意

*   用了with rollup之后，不可以使用order by进行排序。尽管在group by内可以指定列的顺序，但是由于with rollup增加的列仍然会插入相关列的后边。如果对排序有强需求，可以在外围包一层查询进行order by。

*   如果被group by的列如果包含NULL值，那么使用with rollup产生的结果有可能不对。比较好的办法是在使用with rollup之前先把表中的NULL值用某些常量替换。

group\_concat(字段)：统计每个分组指定字段的信息集合，每个信息之间使用逗号进行分隔

## 连接查询

&#x20;如果为表起了别名，则查询的字段就不能用原来的表名去限定

内连接

**内连接是将一张表的每一条记录去另外一张表根据条件匹配。**

1.  匹配成功：保留连接的数据。

2.  匹配失败：都不保留。

外连接

*   **外连接有主表和从表之分**

    1.  左连接，左表为主表

    2.  右连接，右表为主表

*   **外连接是将主表的记录去匹配从表的记录**

    1.  匹配成功保留&#x20;

    2.  匹配失败（全表）：也保留，只是从表字段置空

## 子查询

出现在其它语句中的select语句称为子查询

子查询语句的结果只有一个值时就可以直接使用

如果是一个单列表就用in，any，all作为筛选条件（some是any的别名，用法相同）

如果是一个多列表就通过连接查询来使用

## 分页查询

limit \[offset,]size;

*   offset要显示条目的起始索引（起始索引默认从0开始）

*   size 要显示的条目个数

# 常见约束

*   not null 非空，当设置为非空时，数字类型默认设置为0，字符类型默认设置为空字符串。

*   default 设置默认值

*   unique 唯一，不能重复

*   primary key 主键，非空且不能重复，相当于not null 加unique

*   foreign key 外键，用于保证该字段的值必须来自与主表的关联列的值

```sql
# 创建表时添加外键
create table 表名(
字段1 字段类型1 约束条件1,
字段2 字段类型2 约束条件2,
···,
foreign key(要设外键的字段) references 主表名(要关联的字段)
);

# 为已存在的表增加外键
alter table 表名 add foreign key(要设外键的字段) references 主表名(要关联的字段);
```

由于添加外键约束后不利于后续表的改动和维护，因此现在很少用。一般在代码层约束

*   auto\_increment 自动递增，从上一条数据的值开始递增

# 事务

一个或一组sql语句组成一个执行单元，这个执行单元要么全部执行，要么全部不执行

事务的特性：
原子性：一个事务不可再分割，要么都执行要么都不执行
一致性：一个事务执行会使数据从一个一致状态切换到另外一个一致状态
隔离性：一个事务的执行不受其他事务的干扰
持久性：一个事务一旦提交，则会永久的改变数据库的数据

隐式事务：事务没有明显的开启和结束的标记
比如insert、update、delete语句

显式事务：事务具有明显的开启和结束的标记
前提：必须先设置自动提交功能为禁用

set autocommit=0;

步骤1：开启事务
set autocommit=0;
start transaction;可选的


步骤2：编写事务中的sql语句(select insert update delete)
语句1;
语句2;
...

步骤3：结束事务
commit;提交事务
rollback;回滚事务

# 视图

使用场景

1.  多个地方用到同样的查询结果

2.  该查询结果使用的sql语句较复杂

语法

```sql

```



修改原始表（一个或多个）数据会影响视图，修改视图数据（需要权限）会影响原始表，但是很多情况下不能修改视图数据，例如当视图中有原始表中没有的列时（如通过运算、组合和聚合函数产生的列），因此不推荐修改视图数据

视图在查询时的使用：与普通表几乎一样

删除视图

```sql

```

# 存储过程

一组预先编译好的SQL语句的集合

优点

*   提高代码重用性

*   因为已经预先编译好，调用时不需再次编译，执行效率高

*   传输时一次传输，减少了与服务器的连接次数

创建语法

```sql
create procedure 存储过程名(参数列表)
begin
  sql语句
end
```

1.  参数列表格式：参数模式 参数名 参数类型

    *   参数模式in，out，inout

2.  如果存储过程只有一句话，begin和end可以省略

调用语法

```sql
call 存储过程名();
```

删除存储过程，不能一次删除多个

```sql
deop procedure 存储过程名;
```

查看存储过程信息

```sql
show create procedure 存储过程名;
```

不能修改存储过程中的sql语句，想要修改直接删除新建一个

# 函数

一组预先编译好的SQL语句的集合

优点（与存储过程一样）

*   提高代码重用性

*   因为已经预先编译好，调用时不需再次编译，执行效率高

*   传输时一次传输，减少了与服务器的连接次数

与存储过程的区别

*   存储过程可以没有返回，或有一个至多个返回，适合做批量插入或批量更新

*   函数有且仅有一个返回，适合做处理数据后返回一个结果的任务

语法

```sql
create function 函数名(参数列表) returns 返回类型
begin
  sql语句
end
```

参数列表格式：参数名 参数类型

可以不定义参数，但是不能不定义返回类型

查看函数

```sql
show create function 函数名;
```

删除函数

```sql
drop function 函数名;
```

# 索引

索引在MySQL中也叫做“键”，是存储引擎用于快速找到记录的一种数据结构。

索引对于良好的性能非常关键，尤其是当表中的数据量越来越大时，索引对于性能的影响愈发重要。 &#x20;


索引优化应该是对查询性能优化最有效的手段了。索引能够轻易将查询性能提高好几个数量级。 &#x20;


索引相当于字典的音序表，如果要查某个字，如果不使用音序表，则需要从几百页中逐页去查。

索引的功能就是加速查找


mysql中的primary key，unique等也都是索引，这些索引除了加速查找以外，还有约束的功能。（即主键自带索引，所以查询主键时一般很快）

索引只需要创建、删除和维护，在查询时仍然是用以前的语句，只是查询用时更短

创建索引

```sql
#方法一：创建表时
CREATE TABLE 表名 (
     字段名1  数据类型 [完整性约束条件…],
     字段名2  数据类型 [完整性约束条件…],
     [UNIQUE | FULLTEXT | SPATIAL ]   INDEX | KEY
     [索引名]  (字段名[(长度)]  [ASC |DESC]) 
     );


#方法二：CREATE在已存在的表上创建索引
CREATE  [UNIQUE | FULLTEXT | SPATIAL ]  INDEX  索引名 
ON 表名 (字段名[(长度)]  [ASC |DESC]) ;
 

#方法三：ALTER TABLE在已存在的表上创建索引
ALTER TABLE 表名 ADD  [UNIQUE | FULLTEXT | SPATIAL ] INDEX
索引名 (字段名[(长度)]  [ASC |DESC]) ;
                              
 #删除索引：DROP INDEX 索引名 ON 表名字;
```

## explain

explain可以查看SQL语句有没有使用上了索引，有没有做全表扫描，等等

常用于当查询语句很慢时（是慢查询时），验证是否用到了想用的索引

```sql
# explain 的语法

```

expain出来的信息有10列，分别是id、select\_type、table、type、possible\_keys、key、key\_len、ref、rows、Extra

id：选择标识符 &#x20;

select\_type：表示查询的类型。 &#x20;

table：输出结果集的表 &#x20;

partitions：匹配的分区 &#x20;

type：表示表的连接类型 &#x20;

possible\_keys：表示查询时，可能使用的索引 &#x20;

key：表示实际使用的索引 &#x20;

key\_len：索引字段的长度 &#x20;

ref：列与索引的比较 &#x20;

rows：扫描出的行数(估算的行数) &#x20;

filtered：按表条件过滤的行百分比 &#x20;

Extra：执行情况的描述和说明
